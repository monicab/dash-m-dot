Funny Math With JS

In this first example, you can see type coercion in action. When the Javascript interpreter finds a string, it coverts all other arguments of the addition operation into strings.

    >>> '1' + 2 + 3 // returns '123'
    >>> 3 + 2 + '1' // returns '51'
    >>> 3 + 2 + 1 // returns 6

With this example, we can see that boolean false is coverted to 0 and true is converted to 1.

    >>> false + 0 // returns 0
    >>> true + 0 // returns 1
    >>> false - 0; // returns 0
    >>> true - 0; // returns 1

When I was playing with parseInt, I was curious to see if parseInt(false) = 0 and parseInt(true) = 1 but it's not.

    >>> parseInt('1.0') //returns 1
    >>> parseInt('2.5') //returns 2
    >>> parseInt('1.45abc') //returns 1
    >>> parseInt(false) //returns NaN
    >>> parseInt(true) //returns NaN

    >>> parseFloat('1.5') // returns 1.5
    >>> parseFloat('1.0') // returns 1
    >>> parseFloat('1.45abc') //returns 1.45
    >>> parseFloat('1.50') //returns 1.5
    >>> parseFloat('1.50').toFixed(2) //returns "1.50"

No surprises with Math.round!

    >>> Math.round('1.3') //returns 1
    >>> Math.round('1.5') //returns 2
    >>> Math.round('1.8') //returns 2
    >>> Math.round('1.45abc') //returns NaN

Even though you wouldn't expect 0.1 + 0.2 = 0.30000000000000004, this is actually not a bug but done by design. Check the floating-point arithmetic article. If you are dealing with currency, just do the trick display below. 

    >>> 0.1 + 0.2 // returns 0.30000000000000004
    >>> (0.1*10 + 0.2*10)/10 // returns 0.3

Strings With JS

substring method does NOT include the character in the index passed as the second argument while substr does.

    >>> var message = "San Francisco is a great city!";
    >>> message.substring(1, 4) // return "an "
    >>> message.substr(1, 4)    // returns "an F"
